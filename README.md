# MAI201: Assignment 2: Data Validation & Testing

Validating a messy customer dataset using **Great Expectations** and testing supporting utility functions with **pytest**. No model training is involved - this focuses entirely on the data validation component of an MLOps pipeline.

## Repository Structure

```
Assignment2/
├── customer_data.csv              # Provided messy dataset (5,015 rows)
├── validate.py                    # Great Expectations setup + validation run
├── utils.py                       # Data utility functions (load_csv, clean_phone, validate_email)
├── test_utils.py                  # pytest unit tests for utils.py
├── requirements.txt               # Python dependencies
├── assignment2_report.md          # Full report: results, issues found, reflection
├── screenshots/                   # Evidence screenshots referenced in the report
│   ├── data_quality_issues.png
│   ├── gx_validation_result1.png
│   ├── gx_validation_result2.png
│   └── pytest_execution_result.png
└── gx/                            # Great Expectations project (auto-generated by validate.py)
    ├── expectations/customer_data_expectations.json
    ├── validation_definitions/customer_validation.json
    └── uncommitted/
        ├── data_docs/local_site/index.html   # Generated HTML data quality report
        └── validations/                       # Raw validation run results (JSON)
```

## Setup

1. **Clone the repository**
   ```
   git clone https://github.com/ritikalal911/mlops-assignment2-data-validation.git
   ```

2. **Create and activate a virtual environment** (recommended)
   ```
   python -m venv mlops_a2
   mlops_a2\Scripts\activate      # Windows
   source mlops_a2/bin/activate   # macOS/Linux
   ```

3. **Install dependencies**
   ```
   pip install -r requirements.txt
   ```

## Running the Validation

```
python validate.py
```

This will:
- Load `customer_data.csv`
- Create the `customer_data_expectations` suite with all 9 expectations
- Run validation against the dataset
- Print a full results table (pass/fail, counts, and percentages per check) to the terminal
- Generate the Data Docs HTML report at `gx/uncommitted/data_docs/local_site/index.html`

## Running the Tests

```
pytest test_utils.py -v --cov=utils --cov-report=term-missing
```

This runs all 18 unit tests covering `load_csv()`, `clean_phone()`, and `validate_email()`, and reports test coverage on `utils.py`.
